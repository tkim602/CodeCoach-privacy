Privacy Policy

CodeCoach — Chrome Extension
Last updated: 2026-06-08

CodeCoach is a Chrome extension that helps users practice coding problems by providing progressive AI hints, code analysis, debugging guidance, local study notes, review reminders, and weakness tracking on supported coding-practice pages such as LeetCode and Programmers.

Important AI disclosure: When you request AI feedback, CodeCoach may send your typed question, current editor code, selected text, visible problem context, and run or test-result messages from supported coding-practice pages to OpenAI API. This information is used only to provide the hint, analysis, debugging guidance, note, or chat response you requested.

Summary

  • CodeCoach runs only on supported coding-practice pages and required API endpoints.
  • CodeCoach does not access all websites.
  • CodeCoach does not sell user data.
  • CodeCoach does not use user data for advertising.
  • CodeCoach does not transfer user data to data brokers or information resellers.
  • CodeCoach does not use third-party analytics SDKs or behavioral tracking.
  • Most study data is stored locally in your browser using Chrome extension storage.
  • AI requests are sent directly from your browser to OpenAI using the API key you provide.
  • Optional sign-in and optional cloud sync use Firebase / Google services.

1. Scope of this policy

This Privacy Policy applies only to the CodeCoach Chrome extension. It explains what information CodeCoach may collect, store, transmit, use, or share when you use the extension.

This policy does not apply to third-party websites or services that CodeCoach works with, including LeetCode, Programmers, OpenAI, Firebase, Google, or Chrome itself. Those services are governed by their own privacy policies and terms.

2. Information CodeCoach may collect, store, or process

2.1 Information stored locally in your browser

CodeCoach stores the following information locally using Chrome extension storage:

  • Your OpenAI API key, if you choose to save it in the extension settings
  • Extension settings, such as language, model, feature preferences, and display options
  • Local study notes, wrong-answer notes, and review metadata
  • Hint history, including timestamp, hint level, platform, programming language, and topic tags
  • Weakness-tracking metadata, such as topic category, signal type, confidence score, and review status
  • Code snippets, code snapshots, explanations, or notes that you explicitly save
  • Reminder settings, notification settings, and local review schedule information

Locally stored data remains on your device unless you delete it, uninstall the extension, clear Chrome extension storage, or enable optional cloud sync.

2.2 Information used for AI requests

When you request an AI feature, such as a hint, code analysis, debugging help, note generation, or chat response, CodeCoach may send the following information to OpenAI's API:

  • Your typed question, message, or instruction
  • Your current editor code from the supported coding-practice page
  • Visible problem context from the supported coding-practice page, if included for that request or enabled in settings
  • Selected text, selected code lines, run results, error messages, or test output, when used for the requested feature
  • Basic request metadata, such as platform name, programming language, hint level, and selected AI model

AI requests are sent directly from your browser to OpenAI using your own API key. The CodeCoach developer does not operate a separate backend proxy for OpenAI requests in the current version and does not receive, store, or log the content of your OpenAI requests.

OpenAI processes API requests according to OpenAI's applicable terms and privacy policy: https://openai.com/policies/privacy-policy .

2.3 Information from supported coding-practice pages

CodeCoach is designed to work only on supported coding-practice pages. On those pages, CodeCoach may read or process the following information to provide user-facing study features:

  • Problem title
  • Visible problem description, examples, constraints, or instructions
  • Your current editor code
  • Selected text or selected code lines
  • Visible run results, test results, error messages, or submission-status messages
  • Page URL and platform name, only to detect whether the page is supported

CodeCoach may observe run or submission result messages on supported coding-practice pages to help create debugging feedback, wrong-answer notes, and learning metadata. CodeCoach does not collect cookies, passwords, authentication headers, payment data, or unrelated HTTP payloads.

2.4 Optional Google sign-in information

CodeCoach may offer optional Google sign-in through Firebase Authentication. If you sign in, CodeCoach may process your Google account identifier, such as your email address, display name, profile image URL, and Firebase user ID, for authentication and account-related features.

2.5 Optional cloud sync information

If you enable optional cloud sync, CodeCoach may sync learning metadata to Firebase Firestore under your user ID. Synced metadata may include:

  • User ID generated by Firebase Authentication
  • Hint events and learning events
  • Weakness categories and learning-signal metadata
  • Review schedule metadata
  • Problem identifiers, platform names, programming languages, and timestamps
  • Sync status, settings metadata, and account-related metadata

Cloud sync is optional and can be disabled from the extension settings.

CodeCoach does not sync full platform problem statements, saved code snapshots, or wrong-answer note bodies unless a future version explicitly discloses that behavior and requests the required permission or consent.

3. Information CodeCoach does not collect

CodeCoach does not collect or store the following as developer-controlled data:

  • Full browsing history
  • Data from unsupported websites
  • Page screenshots or screen recordings
  • Keystroke logging data
  • Passwords, cookies, session tokens, or authentication headers
  • Payment information or financial account information
  • Government identification numbers
  • Health information
  • Precise location information
  • Advertising identifiers
  • Data for advertising or behavioral tracking

CodeCoach does not permanently store platform problem statements, examples, constraints, editorial content, contest content, assessment content, or page HTML in any developer-controlled database.

4. How CodeCoach uses information

CodeCoach uses information only to provide and improve the extension's user-facing coding-practice features. Specifically, CodeCoach may use information to:

  • Generate progressive AI hints requested by the user
  • Analyze the user's current code and provide debugging guidance
  • Explain selected code lines or selected text
  • Create local wrong-answer notes or study notes requested by the user
  • Track local weakness categories and learning patterns
  • Schedule optional review reminders and notifications
  • Detect whether the current page is a supported coding-practice page
  • Provide optional Google sign-in and optional cloud sync
  • Maintain extension settings and user preferences
  • Protect against abuse, troubleshoot errors, and comply with legal obligations when necessary

CodeCoach does not use collected or processed information to determine creditworthiness, does not use it for advertising, and does not transfer it to data brokers or information resellers.

5. How CodeCoach shares information

CodeCoach shares information only as necessary to provide the extension's disclosed functionality.

  • OpenAI: When you request an AI feature, CodeCoach sends the necessary prompt, code, problem context, and related request data to OpenAI's API using the API key you provide.
  • Firebase / Google: If you sign in or enable optional cloud sync, CodeCoach uses Firebase / Google services for authentication and cloud-synced learning metadata.
  • Legal or safety reasons: CodeCoach may disclose information if required to comply with applicable law, legal process, or a valid government request, or to protect against fraud, abuse, or security threats.

CodeCoach does not sell user data. CodeCoach does not share user data with advertisers, advertising networks, data brokers, or information resellers.

6. Google API Limited Use disclosure

CodeCoach's use and transfer of information received from Google APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements.

CodeCoach uses Google user data only to provide and improve user-facing authentication and optional sync features. CodeCoach does not use Google user data for advertising, does not sell Google user data, and does not transfer Google user data to third parties except as necessary to provide the extension's disclosed functionality, comply with law, or protect against abuse.

7. Supported pages and website access

CodeCoach does not access all websites. CodeCoach is designed to run only on supported coding-practice pages and required API endpoints.

Supported coding-practice pages may include LeetCode and Programmers problem-practice pages. CodeCoach uses site access to detect the current problem, read visible problem context, read your current editor code, detect selected text, and detect run or test-result messages for user-facing study features.

Required API endpoint access may include OpenAI API for user-requested AI features and Firebase / Google APIs for optional authentication and optional cloud sync.

8. Chrome extension permissions

CodeCoach requests Chrome extension permissions only to provide its stated features. Depending on the installed version, CodeCoach may request permissions such as:

  • storage: Stores settings, local study notes, hint history, weakness categories, and review metadata.
  • sidePanel: Displays the CodeCoach assistant interface next to supported coding-practice pages.
  • scripting: Reads visible problem context and editor content from supported pages for user-requested features.
  • activeTab: Allows CodeCoach to work with the currently active supported coding-practice tab after user interaction.
  • tabs: Detects whether the active tab is a supported problem-practice page.
  • clipboardWrite: Allows users to copy generated hints, explanations, and notes.
  • identity: Supports optional Google sign-in.
  • notifications: Shows optional review reminders or study notifications.
  • alarms: Schedules optional review reminders.

9. Security

CodeCoach transmits external API requests only over HTTPS, including requests to OpenAI API and Firebase / Google APIs. CodeCoach does not send user data over unencrypted HTTP connections.

Locally saved data is stored using Chrome extension storage on the user's device. Chrome extension storage is tied to the user's browser profile. CodeCoach does not add a separate encryption layer to local Chrome storage in the current version, so users should protect their browser profile, operating-system account, and device.

Users should not save an OpenAI API key in CodeCoach on a shared or untrusted device. Users can remove the saved API key from the extension settings at any time.

10. Data retention

Local extension data is kept until you delete it, uninstall the extension, or clear Chrome extension storage.

Optional Firebase learning metadata is kept until you delete it, disable and clear cloud-synced learning data, or request deletion.

OpenAI API requests are retained according to OpenAI's applicable policies for the API account and plan used with your key.

11. Data deletion

You can delete locally stored extension data from the CodeCoach settings or options page where supported. You can also remove local extension data by uninstalling CodeCoach or clearing Chrome extension storage.

If you enabled cloud sync, you can disable sync and delete synced learning metadata from the extension settings where supported. You may also contact the developer using the contact information below for deletion assistance.

12. User access, correction, and deletion requests

You may request access to, correction of, or deletion of developer-controlled data associated with your CodeCoach account by contacting the developer using the email address below.

Local-only data stored in your browser can be viewed, exported where supported, or deleted from the extension settings or by clearing Chrome extension storage.

Deleting certain data, such as settings, API keys, notes, or synced learning metadata, may limit or disable some CodeCoach features.

13. AI feature disclosure

CodeCoach does not automatically send your data to OpenAI merely because you visit a page. AI-related data is sent when you use an AI feature, such as asking a question, requesting a hint, requesting code analysis, requesting debugging help, or generating a note.

When you use an AI feature, the information sent to OpenAI is limited to what is needed to provide the requested response, such as your prompt, current code, selected text, visible problem context, and relevant run or test-result messages.

14. Children

CodeCoach is intended for coding-practice users and is not directed to children under 13. CodeCoach does not knowingly collect personal information from children under 13.
